# 架构检查清单

P3 阶段 Chief Architect / CTO 使用。

## 必检项

- [ ] **架构风格匹配** — 单体/微服务/模块化单体与业务阶段匹配
- [ ] **模块边界清晰** — 无循环依赖，限界上下文明确
- [ ] **接口契约稳定** — 版本化、向后兼容策略
- [ ] **数据库设计** — 满足范式或有显式反范式理由
- [ ] **索引策略** — 匹配查询模式，无全表扫描热点
- [ ] **缓存策略** — 缓存什么 + 失效方案成对存在
- [ ] **一致性边界** — 事务边界明确，强一致 vs 最终一致有选择
- [ ] **水平扩展** — 无单点，无状态耦合
- [ ] **容灾方案** — RTO/RPO 明确，有恢复路径
- [ ] **事件流可靠性** — 异步消息有重试 + 死信机制

## 红线

- [ ] 过早微服务化（团队 <10 人却拆 20 服务）
- [ ] 分布式单体（拆了但耦合还在）
- [ ] 无事务边界的一致性承诺
- [ ] 一个大表装所有数据
- [ ] 缓存无失效方案
