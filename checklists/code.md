# 代码检查清单

P3 阶段 Staff Engineer / QA Lead 使用。

## 必检项

- [ ] **关键路径测试覆盖** — 核心流程有测试，非行覆盖率数字
- [ ] **边界测试** — 空值、超长、并发、时区、负数
- [ ] **异常处理** — 不吞异常，错误可恢复或有降级
- [ ] **并发安全** — 共享状态有锁/无锁方案
- [ ] **复杂度** — 圈复杂度合理，无上帝函数
- [ ] **DRY** — 无明显重复代码块
- [ ] **命名一致性** — 风格统一，命名表意
- [ ] **性能热点** — 关键路径无 N+1 查询、无同步阻塞
- [ ] **内存管理** — 无泄漏，无大对象常驻
- [ ] **测试金字塔** — 单元 > 集成 > E2E 比例合理

## 红线

- [ ] 测试只测 happy path
- [ ] 行覆盖率 >80% 但关键路径无测试
- [ ] 吞异常（catch 后空块）
- [ ] 同步调用外部服务无超时
- [ ] flaky test 长期不修
