# 产品检查清单

P3 阶段 Product Director / UX Expert 使用，P6 隐藏问题阶段复核。

## 必检项

- [ ] **产品定位** — 一句话能说清"为谁解决什么问题"
- [ ] **真需求验证** — 有付费意愿证据，不是"用户说想要"
- [ ] **MVP 边界** — 最小可验证版本，不是最小可发布版本
- [ ] **PMF 信号** — 有无 retention / NPS / 自然增长数据
- [ ] **功能优先级** — 明确什么先做什么后做，有理由
- [ ] **用户反馈闭环** — 收集 → 分析 → 响应的机制存在
- [ ] **新手体验** — 新用户首次价值实现路径（TTV）清晰
- [ ] **学习成本** — 上手时间可量化
- [ ] **认知负担** — 关键路径同时处理的信息量可控
- [ ] **关键路径摩擦** — 核心流程步数合理，无多余跳转

## 红线（命中即记为问题）

- [ ] 没有任何用户验证数据就堆功能
- [ ] 创始人把自己当目标用户
- [ ] 功能数量成为产品好坏标准
- [ ] 无法一句话说清定位
